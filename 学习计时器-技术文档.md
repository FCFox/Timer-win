# 学习计时器技术设计文档

## 1. 技术方案概览

| 范围 | 选型 |
|---|---|
| 开发语言 | Python 3.12 |
| 桌面 UI | PySide6（Qt 6） |
| 图表 | Qt Charts 或自绘 `QPainter` |
| 输入活动检测 | Windows API `GetLastInputInfo`，通过 `ctypes` 调用 |
| 锁屏/电源事件 | Qt Native Event Filter + Windows 消息 |
| 本地数据库 | SQLite 3 |
| 数据访问 | Python `sqlite3` + Repository 模式 |
| 配置 | SQLite 设置表；启动前配置可使用 JSON |
| 日志 | Python `logging` + RotatingFileHandler |
| 测试 | pytest、pytest-qt |
| 代码质量 | Ruff、mypy |
| 打包 | PyInstaller |
| 安装包 | Inno Setup |

选择 PySide6 的原因是它具备成熟的 Windows 桌面能力、原生系统托盘、高清缩放支持和丰富的自定义绘制能力，适合制作现代化界面。

## 2. 关键技术决策

### 2.1 输入活动检测

程序不安装键盘钩子，也不捕获具体输入事件。后台每秒调用一次 Windows `GetLastInputInfo`，读取系统最后一次键盘或鼠标输入距当前的时间。

优点：

- 不需要管理员权限。
- 不记录任何输入内容。
- 系统开销低。
- 即使应用在后台也可工作。

注意：API 返回的是系统级输入时间，无法区分输入是否与学习有关。因此产品中的“工作”准确含义是“电脑处于有输入活动的状态”。

### 2.2 状态区间而非逐秒数据

数据库只保存连续状态区间，例如 `09:00～09:45 work`，而不是每秒写入一条记录。这可以显著减少数据库体积和写入频率。

状态发生变化、程序暂停、系统锁屏、跨日或程序退出时关闭当前区间并创建新区间。运行期间每 30 秒保存一次检查点，降低异常退出造成的数据损失。

### 2.3 单调时钟与墙上时间

- 状态持续时长使用 `time.monotonic()` 计算，避免用户修改系统时间造成跳变。
- 数据库存储使用带时区的 UTC 时间戳。
- 界面展示时转换为本地时间。
- 跨日分割按照本地时区执行。

## 3. 软件架构

采用分层架构，UI 不直接访问 Windows API 或执行 SQL。

```text
Presentation（PySide6 页面、组件、ViewModel）
              ↓ 信号/命令
Application（计时服务、统计服务、设置服务）
              ↓
Domain（状态机、时间区间、统计规则）
              ↓
Infrastructure（Windows API、SQLite、日志、开机启动）
```

核心原则：

- 计时逻辑独立于界面，可进行纯单元测试。
- Windows 特有能力封装在基础设施层。
- UI 通过 Qt 信号接收状态与统计快照。
- 数据库操作在独立工作线程中执行，避免阻塞界面。

## 5. 领域模型与状态机

### 5.1 状态定义

```python
from enum import StrEnum

class ActivityState(StrEnum):
    WORKING = "working"
    IDLE = "idle"
    PAUSED = "paused"
    UNTRACKED = "untracked"
```

### 5.2 状态优先级

状态按以下优先级决定：

1. 用户手动暂停 → `PAUSED`
2. 系统锁屏、休眠或正在关机 → `UNTRACKED`
3. 最后输入时间小于空闲阈值 → `WORKING`
4. 其他情况 → `IDLE`

### 5.3 状态转换

| 当前状态 | 事件 | 新状态 |
|---|---|---|
| 未记录 | 首次输入 | 工作 |
| 工作 | 无输入达到阈值 | 空闲 |
| 空闲 | 任意键鼠输入 | 工作 |
| 工作/空闲 | 用户暂停 | 暂停 |
| 暂停 | 用户恢复且最近有输入 | 工作 |
| 暂停 | 用户恢复且已超过阈值 | 空闲 |
| 工作/空闲 | 锁屏或休眠 | 未记录 |
| 未记录 | 解锁或唤醒后产生输入 | 工作 |

从工作进入空闲时，需要把转换点回溯到“最后输入时间 + 空闲阈值”，而不是检测到转换的轮询时刻。

## 6. 计时算法

每 1 秒执行一次检测：

```text
读取当前单调时间
读取 Windows 最后输入时长 idle_seconds
读取用户暂停状态与系统会话状态
根据优先级计算 desired_state

如果 desired_state 与 current_state 不同：
    计算准确的状态边界
    关闭当前数据库区间
    创建新的状态区间
    发出 state_changed 信号

更新今日聚合快照
必要时写入检查点
发出 snapshot_updated 信号
```

为了正确处理空闲阈值，建议计时服务保留：

- `current_state`
- `current_segment_started_at`
- `last_input_wall_time`
- `last_input_monotonic_time`
- `last_checkpoint_at`
- `is_session_available`
- `is_user_paused`

## 7. Windows API 封装

`GetLastInputInfo` 的最小封装示意：

```python
from ctypes import Structure, byref, sizeof, windll
from ctypes.wintypes import DWORD, UINT

class LASTINPUTINFO(Structure):
    _fields_ = [("cbSize", UINT), ("dwTime", DWORD)]

def get_idle_milliseconds() -> int:
    info = LASTINPUTINFO()
    info.cbSize = sizeof(LASTINPUTINFO)
    if not windll.user32.GetLastInputInfo(byref(info)):
        raise OSError("GetLastInputInfo failed")
    current_tick = windll.kernel32.GetTickCount()
    return (current_tick - info.dwTime) & 0xFFFFFFFF
```

正式实现建议使用 `GetTickCount64` 处理长时间开机，并为 API 失败增加日志、重试和降级状态。Windows 会话变化可监听 `WM_WTSSESSION_CHANGE`，电源变化可监听 `WM_POWERBROADCAST`。

## 8. 数据库设计

### 8.1 `activity_segments`

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| state | TEXT | working / idle / paused / untracked |
| start_utc | TEXT | ISO 8601 UTC 时间 |
| end_utc | TEXT NULL | 当前区间为空 |
| duration_seconds | INTEGER | 已确认时长 |
| source | TEXT | automatic / manual / system |
| note | TEXT NULL | 可选备注 |
| created_at_utc | TEXT | 创建时间 |

索引：`start_utc`、`state`，以及 `(start_utc, end_utc)`。

### 8.2 `daily_summaries`

| 字段 | 类型 | 说明 |
|---|---|---|
| local_date | TEXT PK | 本地日期 YYYY-MM-DD |
| working_seconds | INTEGER | 工作秒数 |
| idle_seconds | INTEGER | 空闲秒数 |
| paused_seconds | INTEGER | 暂停秒数 |
| untracked_seconds | INTEGER | 未记录秒数 |
| goal_seconds | INTEGER | 当日目标快照 |
| updated_at_utc | TEXT | 更新时间 |

该表是可重建的缓存，真实数据以 `activity_segments` 为准。

### 8.3 `settings`

| 字段 | 类型 | 说明 |
|---|---|---|
| key | TEXT PK | 配置键 |
| value_json | TEXT | JSON 编码值 |
| updated_at_utc | TEXT | 更新时间 |

主要配置包括 `idle_threshold_seconds`、`daily_goal_seconds`、`theme`、`launch_at_login`、`minimize_to_tray` 和 `week_start_day`。

### 8.4 数据一致性

- SQLite 启用 WAL 模式。
- 状态切换时使用事务关闭旧区间并创建新区间。
- 启动时检查未关闭区间；异常退出后的间隔默认标为 `untracked`。
- 数据库 schema 使用版本号和顺序迁移脚本。

## 9. UI 实现方案

### 9.1 页面

- `TodayPage`：今日数据、目标进度、时间轴、当前状态。
- `StatisticsPage`：周/月趋势和工作占比。
- `HistoryPage`：按天查看状态区间并添加备注。
- `SettingsPage`：计时、外观、启动、数据管理设置。

### 9.2 自定义组件

- `StatusPill`：带文字的彩色状态标签。
- `ProgressRing`：目标完成进度环。
- `MetricCard`：指标卡片。
- `ActivityTimeline`：使用 `QPainter` 绘制状态时间轴。
- `WeeklyBarChart`：周学习时长柱状图。

### 9.3 主题

- 使用 QSS 与 Qt Palette 统一主题。
- 颜色、间距、圆角、字号集中为 design tokens。
- 图标使用同一套 SVG 线性图标。
- 所有尺寸使用布局管理器，避免绝对坐标。
- 开启 Qt 高 DPI 支持，测试 100%、125%、150% 和 200% 缩放。

## 10. 线程与通信

- UI 线程：窗口渲染与用户交互。
- 计时器：使用 UI 线程中的轻量 `QTimer` 触发检测，Windows API 调用必须快速返回。
- 数据库工作线程：处理查询、聚合和批量导出。
- 服务通过 Qt Signal/Slot 与页面通信。
- 禁止从后台线程直接修改控件。

短小的状态写入可以同步执行；若实测出现 UI 卡顿，再统一迁移到数据库工作线程。首选保持设计简单并用性能测试验证。

## 11. 系统托盘与单实例

- 使用 `QSystemTrayIcon` 提供显示主窗口、暂停/恢复和退出菜单。
- 点击窗口关闭按钮默认最小化到托盘。
- 真正退出前关闭当前区间并执行数据库 checkpoint。
- 使用 `QLocalServer`/`QLocalSocket` 实现单实例；第二次启动时唤醒已有窗口。
- 开机启动使用当前用户注册表 `HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run`，无需管理员权限。

## 12. 异常场景处理

| 场景 | 处理方式 |
|---|---|
| 程序异常退出 | 下次启动关闭遗留区间，缺失部分标为未记录 |
| 系统休眠 | 收到电源事件时关闭当前区间；唤醒后等待输入 |
| 锁屏 | 切换为未记录，解锁后重新判断 |
| 系统时间被修改 | 持续时长使用单调时钟，记录异常日志 |
| 跨越午夜 | 在本地 00:00 分割区间并分别聚合 |
| 数据库被锁定 | 短暂重试，失败后展示非阻塞警告并写错误日志 |
| 数据库损坏 | 保留原文件，提示从备份恢复或创建新库 |
| API 调用失败 | 暂时标为未记录，不猜测用户状态 |

### 13.3 UI 测试

## 14. 安全与隐私

- 不使用低级键盘钩子，不申请管理员权限。
- 日志不得包含输入内容、窗口标题或个人文件路径。
- 导出和删除由用户主动触发。
- 删除全部数据需二次确认。
- 安装包和可执行文件发布时进行哈希校验；正式发布建议使用代码签名证书。

## 16. 开发顺序

1. 建立领域模型与可测试状态机。
2. 实现 Windows 输入、会话和电源适配器。
3. 实现 SQLite 区间仓储和异常恢复。
4. 完成今日页与系统托盘，形成可用 MVP。
5. 增加统计页、主题、导出和安装包。
6. 完成性能测试、兼容性测试和隐私检查。

第一项可交付里程碑应是一个无完整视觉皮肤但能可靠后台计时、重启恢复和显示今日数据的版本；随后再完成视觉与统计功能，降低核心计时逻辑被 UI 开发掩盖的风险。
