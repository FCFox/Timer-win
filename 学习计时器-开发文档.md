# 学习计时器开发文档

## 1. 项目概述

### 1.1 项目名称

暂定名称：**StudyFlow 学习计时器**

### 1.2 产品定位

StudyFlow 是一款运行于 Windows 的轻量桌面学习计时工具。程序根据用户的键盘或鼠标活动自动判断“工作”和“空闲”状态，帮助用户了解真实投入时间、空闲分布和每日学习节奏。

产品强调三个特点：

- 自动记录：无需反复点击开始、暂停。
- 简洁美观：适合常驻桌面和日常使用。
- 隐私优先：只读取系统最后输入时间，不记录按键内容、鼠标位置、窗口内容或浏览历史。

## 2. 产品目标

### 2.1 核心目标

- 自动统计每天的工作时长与空闲时长。
- 用时间轴直观展示一天中的状态变化。
- 提供日、周、月维度的学习统计。
- 支持学习目标和提醒，辅助用户改进学习规划。
- Windows 开机后可自动运行，并在系统托盘中常驻。

### 2.2 首版不包含

- 多设备同步和账号系统。
- 云端数据存储。
- 监控输入的具体内容。
- 网站、软件或文档级别的活动追踪。
- 团队监督和排行榜。

## 3. 目标用户与场景

目标用户包括学生、备考人员、自学者和居家办公人员。

典型场景：

1. 用户开始电脑学习，程序自动将持续键鼠活动计入工作时间。
2. 用户离开电脑，超过设定阈值后，后续时间自动计入空闲。
3. 用户回到电脑并产生输入，状态立即恢复为工作。
4. 用户在晚上查看时间轴、有效学习时长和目标完成率。
5. 用户按周复盘每天的学习投入与空闲分布。

## 4. 状态判定规则

### 4.1 基本定义

- **工作状态**：系统检测到最近存在键盘或鼠标输入。
- **空闲状态**：距离最后一次输入超过空闲阈值。
- **未记录状态**：程序未运行、用户暂停记录，或设备休眠期间。

### 4.2 推荐默认规则

- 默认空闲阈值：5 分钟，可在 1～30 分钟间调整。
- 启动后第一次检测到输入时开始当天记录。
- 输入发生后立即进入工作状态。
- 连续无输入达到阈值后，将这段阈值时间保留为工作时间，阈值之后计为空闲。
- 系统锁屏、休眠或关机期间不自动计为空闲，以避免夸大空闲数据。
- 用户可手动暂停记录；暂停时间显示为“未记录”。

示例：用户 10:00 最后一次输入，阈值为 5 分钟，10:12 再次输入，则 10:00～10:05 为工作，10:05～10:12 为空闲。

## 5. 功能需求

### 5.1 MVP（第一阶段）

| 模块 | 功能 | 验收标准 |
|---|---|---|
| 自动计时 | 检测全局键鼠活动 | 无需窗口聚焦即可正确更新状态 |
| 状态切换 | 工作、空闲、暂停 | 切换准确，界面在 2 秒内更新 |
| 今日概览 | 今日工作、空闲、目标进度 | 数据实时更新且重启后保留 |
| 今日时间轴 | 分色展示状态区间 | 可识别工作、空闲和未记录区间 |
| 手动控制 | 暂停、恢复、结束今日记录 | 操作后立即生效 |
| 设置 | 空闲阈值、每日目标、开机启动 | 修改后持久化并在下次启动生效 |
| 系统托盘 | 最小化、显示、暂停、退出 | 关闭主窗口后可继续后台计时 |
| 本地存储 | 保存状态区间和配置 | 异常退出后不丢失已落盘数据 |

### 5.2 第二阶段

- 周视图和月视图。
- 每日目标完成率、连续达标天数。
- 专注时段分布和趋势图。
- 学习记录备注。
- CSV 数据导出与本地备份。
- 空闲返回时询问是否将部分空闲时间修正为工作。
- 深色/浅色主题和主题色选择。

### 5.3 可选增强

- 番茄钟模式。
- 学习科目或项目分类。
- Windows 通知提醒。
- 数据库加密和可选备份密码。

## 6. 界面与交互设计

### 6.1 视觉方向

- 风格：现代、安静、轻量，避免复杂表格感。
- 主色：靛蓝或蓝紫色；工作状态使用绿色，空闲状态使用暖橙色，未记录使用灰色。
- 圆角卡片、柔和阴影、充足留白。
- 支持浅色和深色模式。
- 默认窗口建议为 1040 × 680，最小尺寸 860 × 560。

### 6.2 主界面布局

左侧为窄导航栏，包含“今日”“统计”“记录”“设置”；右侧为内容区。

“今日”页面包含：

1. 顶部欢迎语、日期和当前状态指示灯。
2. 今日工作时长大卡片，显示目标进度环。
3. 空闲时长、工作占比、最长连续工作时段三个摘要卡片。
4. 今日时间轴，绿色代表工作、橙色代表空闲、灰色代表未记录。
5. 最近状态切换列表。
6. 暂停/恢复主按钮。

### 6.3 状态反馈

- 工作：绿色圆点和“正在学习”。
- 空闲：橙色圆点和“已空闲 X 分钟”。
- 暂停：灰色圆点和“记录已暂停”。
- 鼠标悬停托盘图标时显示今日工作时长和当前状态。

### 6.4 易用性

- 关键操作不超过两次点击。
- 退出程序时明确提示“退出后将停止记录”。
- 首次启动提供 3 步引导：自动记录原理、隐私说明、目标与阈值设置。
- 所有统计项提供简短解释，避免“工作占比”等指标产生歧义。

## 7. 数据与隐私

- 所有数据默认仅保存在本机。
- 不记录具体按键、输入文本、鼠标坐标、屏幕画面和活动窗口标题。
- 不访问摄像头、麦克风或浏览器历史。
- 首次启动明确展示采集范围。
- 用户可导出或永久删除全部数据。

## 8. 非功能需求

- 计时误差：正常运行时每日累计误差小于 5 秒。
- CPU 占用：后台稳定运行时目标低于 1%。
- 内存占用：目标低于 150 MB。
- 启动时间：普通 Windows 电脑上目标低于 3 秒。
- 稳定性：数据库写入具有事务保护；异常退出后可恢复。
- 兼容性：优先支持 Windows 10 22H2 与 Windows 11。
- 可访问性：文字和背景对比度符合 WCAG AA 的主要要求；状态不能只依赖颜色表达。

## 9. 开发计划

### 阶段一：基础框架（2～3 天）

- 建立项目结构、配置管理和日志系统。
- 搭建 PySide6 主窗口、导航和主题。
- 实现 SQLite 初始化与数据访问层。

### 阶段二：核心计时（3～4 天）

- 接入 Windows 最后输入时间检测。
- 实现工作、空闲、暂停状态机。
- 处理锁屏、休眠、唤醒和跨日。
- 添加计时单元测试。

### 阶段三：主要界面（4～6 天）

- 完成今日仪表盘、进度环和时间轴。
- 完成设置页和系统托盘。
- 增加浅色/深色主题。

### 阶段四：统计与发布（3～5 天）

- 完成周统计、导出和备份。
- 性能、异常恢复和 UI 测试。
- 使用 PyInstaller 打包安装包。

预计 MVP 开发周期：个人开发约 2～3 周。

## 10. 验收清单

- 程序可以在 Windows 10/11 正常安装和启动。
- 键盘或鼠标输入可以使状态恢复为工作。
- 无输入达到阈值后自动进入空闲。
- 锁屏和休眠时间不会错误计入空闲。
- 暂停、恢复和退出逻辑符合预期。
- 跨日后统计正确归入对应日期。
- 程序重启后历史记录和设置仍存在。
- 托盘常驻和开机启动可正常启用、关闭。
- 主界面在 100% 与 150% 缩放下无明显错位。
- 用户可以清空数据，且清空操作需要二次确认。

## 11. 待确认项

以下内容不阻塞 MVP，可在原型完成后确定：

- “工作时间”是否需要改名为“学习时间”。
- 系统锁屏时间应归入“未记录”还是单独的“离开”。
- 是否需要按科目分类统计。
- 是否允许用户手动修正历史区间。
- 首版是否同时发布免安装版与安装版。
